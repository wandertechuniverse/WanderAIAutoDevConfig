**Architecture Rules:**
1. **Separation of Concerns:** The CLI UI (`bin/cli.js`) must be completely decoupled from the core MCP and Routing logic. 
2. **Schema Validation First:** Implement `zod` to validate all environment variables, API keys, and the `agents_config.json` file. If validation fails, provide a human-readable terminal error and exit cleanly (no ugly stack traces).

4. **Resilient Routing:** The core API client must include a retry mechanism and provider fallback (e.g., wrap the API calls in a try/catch that falls back to Gemini/OpenRouter on a 429 status).
