# Context & Role
You are an expert Full-Stack Next.js Developer and Technical Writer. I am building the **Wander AI Auto Dev Config Hub**, a central web portal for an autonomous multi-agent AI orchestration system.

The application has two primary sections:
1. **The Interactive Agent Chat (`/`)**: A responsive web interface to chat with 15 specialized local AI agents.
2. **The Documentation Hub (`/docs`)**: A beautiful, interactive guide explaining how to use our CLI, configure our MCP server, and deploy the system.

# Tech Stack & Aesthetics
- **Framework**: Next.js (App Router)
- **Styling**: Tailwind CSS, `lucide-react` icons.
- **Aesthetics**: Premium dark mode (`bg-slate-950`), deep navy to vibrant cyan gradients, sleek glassmorphism effects (`backdrop-blur-md`), and highly responsive.
- **LLM Logic**: Multi-provider API handling (OpenAI, @google/genai for Gemini, OpenRouter) reading local `.agent.md` files.

# Task 1: Onboarding & IDE Selection
Create an IDE selection screen (or modal) for first-time visitors with the title "Select an IDE Agent".
- Options must include: **VSCode**, **Cursor**, **Windsurf**, and **Antigravity**.
- Use a sleek, glassmorphic card design.

# Task 2: Responsive Chat Interface (`app/page.tsx`)
Refactor the main chat interface to be perfectly mobile-responsive:
- **Desktop**: Fixed left sidebar listing available agents (read from `data/agents_config.json`), right-side chat window.
- **Mobile**: A glassmorphic top-bar with a hamburger menu. Tapping it triggers a smooth slide-out off-canvas sidebar with a dark overlay. The input box must stay firmly pinned to the bottom.

# Task 3: The Multi-Provider API (`app/api/chat/route.ts`)
Write the API route to handle the chat execution.
1. Accept `agentId` and `messages`.
2. Read the corresponding `data/agents/{agentId}.agent.md` file using `fs/promises`.
3. Check the `ACTIVE_PROVIDER` from `.env` (supports `openai`, `gemini`, `openrouter`, `nvidia`).
4. Route the LLM request to the correct SDK and return the streamed or awaited response.
