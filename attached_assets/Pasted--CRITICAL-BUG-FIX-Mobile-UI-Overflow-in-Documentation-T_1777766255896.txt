# CRITICAL BUG FIX: Mobile UI Overflow in Documentation

The code blocks inside our documentation pages are breaking the mobile layout by forcing the screen width to expand horizontally. We need to strictly trap the width inside the `DocCard` component.

## Task 1: Update the `DocCard` Component
Please completely replace the current `DocCard` component (likely in `components/DocCard.tsx` or similar) with the following code. Pay special attention to the `min-w-0` and `overflow-x-auto` classes, as these are the critical Tailwind hacks needed to stop flex children from expanding past the viewport:

```tsx
"use client";
import { useState } from "react";
import { ChevronDown } from "lucide-react";

export function DocCard({ title, badge, dotColor = "bg-cyan-400", defaultOpen = false, children }) {
  const [isOpen, setIsOpen] = useState(defaultOpen);

  return (
    <div className="w-full max-w-full rounded-xl border border-slate-800/60 bg-slate-900/40 mb-4 overflow-hidden">
      <button 
        onClick={() => setIsOpen(!isOpen)} 
        className="flex w-full items-center justify-between p-4 focus:outline-none hover:bg-slate-800/40 transition-colors"
      >
        <div className="flex items-center text-left min-w-0">
          <div className={`w-1.5 h-1.5 rounded-full shrink-0 ${dotColor}`}></div>
          <h3 className="ml-3 font-semibold text-slate-200 truncate">{title}</h3>
          {badge && <span className="font-mono text-[10px] text-slate-500 ml-2 hidden sm:inline-block shrink-0">{badge}</span>}
        </div>
        <ChevronDown className={`w-4 h-4 text-slate-500 shrink-0 transition-transform duration-200 ${isOpen ? "rotate-180" : ""}`} />
      </button>
      
      <div className={`transition-all duration-300 ease-in-out ${isOpen ? "max-h-[2000px] opacity-100" : "max-h-0 opacity-0"}`}>
        <div className="p-4 pt-0 border-t border-slate-800/30 mt-2 min-w-0 w-full">
          <div className="w-full overflow-x-auto pb-2">
            {children}
          </div>
        </div>
      </div>
    </div>
  );
}
